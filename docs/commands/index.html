Command Reference

Complete guide to all palm commands — v1.2.0.

Installing Tools

Install one or more AI tools. palm auto-detects the best backend.

# Install a single tool $ palm install aider # Install multiple tools in parallel $ palm install claude-code aider ollama # Install sequentially (disable parallel) $ palm install --seq claude-code aider

Running Tools

palm run launches a tool with all relevant vault keys injected as environment variables.

# Run with vault key injection $ palm run aider # Pass arguments through $ palm run aider --model gpt-4o # Run any tool (not just registered ones) $ palm run python my_script.py
How it works: palm reads the tool's required keys from the registry, pulls them from the vault, and injects them as env vars. Your keys never touch disk as plaintext.

Tool Pipelines

Chain AI tools together by piping output. Vault keys are injected across the entire pipeline.

# Pipe code through an AI for review $ palm pipe "cat main.py" "|" "ollama run llama3.3 'review this code'" # Verbose mode shows each step $ palm pipe -v "cat README.md" "|" "aider --message 'improve this'" [1/2] cat: README.md (2.1KB) [2/2] aider: processing... ‚úì Pipeline complete in 12.3s

AI Squad

Run the same task through multiple AI tools in parallel. Four modes:

ModeBehavior
raceFirst successful tool wins (fastest response)
allShow all outputs side by side for comparison
voteAll tools run, a judge AI picks the best result
mergeAll tools run, a judge AI synthesizes all results
# Race mode $ palm squad "explain quicksort" --tools ollama,mods --mode race # Vote with judge $ palm squad "fix this bug" --tools aider,codex --judge ollama --mode vote # Merge all outputs $ palm squad "write tests" --tools aider,codex,claude-code --judge ollama --mode merge

Compose Workflows

Define reusable multi-tool AI workflows in TOML files. Like Docker Compose for AI.

# Create a sample workflow $ palm compose init Created .palm-compose.toml # Run the workflow $ palm compose # Dry run $ palm compose --dry-run

Workflow File Format

# .palm-compose.toml name = "code-review" [[steps]] name = "read-code" run = "cat src/main.py" [[steps]] name = "ai-review" tool = "ollama" args = ["run", "llama3.3", "Review this code:"] input = "step:read-code" depends_on = ["read-code"]

Managing Keys

palm stores API keys in macOS Keychain (macOS) or AES-256-GCM encrypted file (Linux/Windows). Keys are never stored as plaintext.

# Add a key (prompts for value) $ palm keys add ANTHROPIC_API_KEY # List stored keys (values masked) $ palm keys list ANTHROPIC_API_KEY sk-ant-*********** OPENAI_API_KEY sk-*************** # Export as shell statements $ palm keys export # Shell integration (add to shell profile) $ eval "$(palm keys env)" # Remove a key $ palm keys rm OPENAI_API_KEY

LLM Proxy

Local reverse proxy that routes API requests to any LLM provider. Auto-injects keys and logs requests.

# Start in background $ palm proxy start --bg # Point tools at the proxy $ export OPENAI_BASE_URL=http://localhost:4778/openai/v1 $ export ANTHROPIC_BASE_URL=http://localhost:4778/anthropic/v1 # View logs $ palm proxy logs # Stop $ palm proxy stop

Local LLM Server

Orchestrate local LLM runtimes with GPU detection. Delegates to ollama, llama.cpp, or vllm.

# Start local LLM server $ palm serve start Detected GPU: Apple M2 Pro (Metal) Starting ollama... ‚úì Server running on :11434 # Pull a model $ palm serve pull llama3.3 # List available models $ palm serve models # Status $ palm serve status # Stop $ palm serve stop

Git Worktrees

Manage git worktrees to work on multiple branches with AI tools simultaneously.

# Create worktree for a branch $ palm worktree add feature-auth # List active worktrees $ palm worktree list # Run an AI tool inside a worktree $ palm worktree run feature-auth aider "add login form" # Clean up $ palm worktree remove feature-auth
Key feature: palm worktree run launches any AI tool inside the worktree directory with vault keys injected. Your current branch stays untouched.

Knowledge Graph

Encrypted knowledge graph for AI tool memory. Stored as graph.enc using AES-256-GCM — same encryption pattern as the key vault. Designed for AI tools (Claude Code, aider, etc.) to use as persistent memory via CLI.

# Create entities $ palm graph add "Mohamed Salah" --type person $ palm graph add palm --type project # Add observations (facts) $ palm graph observe "Mohamed Salah" "GitHub: msalah0e" $ palm graph observe palm "AI tool manager written in Go" # Create directed relations $ palm graph relate "Mohamed Salah" created palm # Show entity with connections (visual tree) $ palm graph show "Mohamed Salah" ‚óè Mohamed Salah ‚îÇ person ‚îÇ "GitHub: msalah0e" ‚îÇ ‚îî‚îÄ‚îÄ created ‚îÄ‚îÄ palm project # JSON output for AI tools $ palm graph show "Mohamed Salah" --json # Search across names, types, and observations $ palm graph search project $ palm graph search project --json # List all entities (with optional type filter) $ palm graph list $ palm graph list --type person

Graph Visualization

Open an interactive, Obsidian-like force-directed graph view in your browser. Nodes are colored by type, sized by observation count, with directed edge arrows and labels.

# Open in browser $ palm graph view ‚úì Opened graph visualization (12 entities, 8 relations) # Export as standalone HTML file $ palm graph export --format html > my-graph.html # Export as Graphviz DOT (for custom rendering) $ palm graph export --format dot | dot -Tpng -o graph.png
Features: Drag nodes to rearrange, scroll to zoom, search to highlight, hover for details. All data is embedded in a single HTML file — no external dependencies.

TUI Navigator

# Launch interactive project browser $ palm ui # Scan a specific directory $ palm ui ~/Projects

Scans for projects containing .palm.toml, go.mod, package.json, pyproject.toml, etc. Shows installed tools and key status per project.

Configuration

Global Config: ~/.config/palm/config.toml

[parallel] enabled = true concurrency = 4 [install] prefer_uv = true [vault] backend = "auto" # auto | keychain | file

Project Config: .palm.toml

[workspace] name = "my-project" tools = ["aider", "claude-code"] keys = ["ANTHROPIC_API_KEY"]

Project config overrides global config. palm searches up from your current directory to find .palm.toml.

Custom Plugins

Add custom tools by creating TOML files in ~/.config/palm/plugins/.

# ~/.config/palm/plugins/my-tools.toml [[tools]] name = "my-tool" display_name = "My Custom Tool" description = "A custom AI tool" category = "coding" [tools.install] pip = "my-tool" [tools.keys] required = ["MY_API_KEY"]
