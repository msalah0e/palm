CI/CD Integration

Use palm in your CI/CD pipelines to ensure consistent AI tool setup across environments.

GitHub Actions

# .github/workflows/ai-tools.yml name: AI Tools on: [push] jobs: setup: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 # Install palm - name: Install palm run: curl -fsSL https://msalah0e.github.io/palm/install.sh | sh # Install workspace tools - name: Install AI tools run: palm workspace install # Run evaluation - name: Run eval env: OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }} run: palm eval run

Using Workspace Config

Commit .palm.toml to your repo so CI uses the same tools as your team:

# .palm.toml [workspace] name = "my-project" tools = ["aider", "promptfoo"] keys = ["OPENAI_API_KEY"]
# In CI, install all pinned tools $ palm workspace install

Offline CI

For air-gapped CI environments, pre-cache tools:

# On a machine with internet access $ palm cache fetch aider promptfoo $ palm cache bundle ci-tools.tar.gz # In CI (no internet required) $ tar xzf ci-tools.tar.gz -C ~/.cache/palm/ $ palm --offline workspace install
