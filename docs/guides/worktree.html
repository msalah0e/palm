Git Worktrees

Work on multiple branches simultaneously with AI tools using palm worktree.

Why Worktrees?

When using AI tools on a feature branch, you often need to check another branch — review a PR, fix a hotfix, or experiment. Git worktrees let you have multiple branches checked out at once.

Creating a Worktree

# Create a worktree for an existing branch $ palm worktree add feature-auth âœ“ Worktree created at ../myrepo-feature-auth # Specify a custom path $ palm worktree add feature-auth --path ~/worktrees/auth

If the branch doesn't exist, palm creates it from the current HEAD.

Listing Worktrees

$ palm worktree list Path Branch HEAD /Users/me/myrepo main abc1234 /Users/me/myrepo-feature-auth feature-auth def5678

Running Tools in Worktrees

The key feature: run any AI tool inside a worktree without leaving your current branch.

# Run aider in the feature-auth worktree $ palm worktree run feature-auth aider "add login form" # Run claude-code in a worktree $ palm worktree run feature-auth claude-code
Key injection: palm worktree run injects vault keys just like palm run. The tool runs in the worktree directory with full vault access.

Removing Worktrees

$ palm worktree remove feature-auth âœ“ Worktree removed

Typical Workflow

# You're on main, need to work on a feature $ palm worktree add feature-auth # Run an AI tool in that worktree $ palm worktree run feature-auth aider "implement JWT auth" # Meanwhile, fix a hotfix on main (your current branch) $ palm run aider "fix the login bug" # Done with the feature worktree $ palm worktree remove feature-auth
